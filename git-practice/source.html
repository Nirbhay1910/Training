Git Practice

  • git status: shows status
  • git log: gives all previous commit details
  • git diff: gives diff between working directory and staged directory
  • git add --a / git add .: add all the modified files in staging area
  • git commit -m "" : used for commiting with message
  • git clone: used for cloningh a remote repository
  • git commit -a -m "" : Direct commit skipping staging part but only for tracke files
  • git rm filePath : Remove file and added automatically
  • git mv filepath1 filepathWithNewName : Renames file
  • git rm --cached file : Remove already tracking file from git (Do it after putting file name in gitignore)
  • git log -p -n: Shows what last n persons has commited with what files changed
  • git log --stat : will show what person has changed in short
  • git log --pretty=oneline : shows commits in one line
  • git log --since=2.days : shows commits of last 2 days
  • git log --pretty=format:"%h -- %an" : will give hashed commit and author name
  • git commit --amend : With this we can change last commit's message and can merge our staged changes with the last commit (After opening of editor press i to type after that press esc,colon,wq and enter)
  • git restore --staged filepath: Removes file from stage area
  • git checkout -- filePath : Restores the file to previous commit (will only work if file is not in staged area)
  • git checkout -f : Will make everything as previous commit
  • git remote add origin url : "origin" is the name of the url that we gave for easy access
  • git config --global alias.st status : with this git st will work as git status
      My aliases:
    • st: status
    • unstage: restore --staged --
    • last: log -p -1
  • git checkout -b newBranchName : Makes new branch
